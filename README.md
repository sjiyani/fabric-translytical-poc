# Translytical POC вҖ” Microsoft Fabric Write-Back Dashboard

A proof-of-concept implementation of **Microsoft Fabric Translytical Task Flow** вҖ” a pattern that allows users to write data back to a database directly from a Power BI report embedded in Fabric, using User Data Functions (UDF) with zero credential configuration.

> **Reference**: [MS Learn вҖ” Translytical Task Flow Overview](https://learn.microsoft.com/en-us/power-bi/create-reports/translytical-task-flow-overview)  
> **Official Pattern**: [Sujata994 GitHub Gist](https://gist.github.com/Sujata994/c354ec8d0821e875e45c86f2bd1d5cc8)

---

## What This POC Demonstrates

| Capability | Description |
|---|---|
| **Read from DB** | Power BI report reads product data via DirectQuery from Fabric SQL Database |
| **Write back from report** | User clicks a button вҶ’ UDF writes to SQL DB вҶ’ report auto-refreshes |
| **Status update (Tutorial 2)** | Change product status + log to history table |
| **Teams notification** | Request status update triggers an Adaptive Card via Teams webhook |
| **Annotation (Tutorial 1)** | Add free-text annotations to any product |
| **Price update (Tutorial 1)** | Update unit price with full audit history |

## Setup Guide

### Step 1 вҖ” Authenticate

```powershell
az login
```

### Step 2 вҖ” Install Python dependencies

```powershell
python -m venv .venv
.venv\Scripts\Activate.ps1
pip install azure-identity requests pyodbc
```

### Step 3 вҖ” Configure workspace ID

Open `setup_fabric_sql.py` and `create_dashboard_v3.py` and update:

```python
WORKSPACE_ID = "<your-fabric-workspace-id>"
```

Your workspace ID is in the URL when you open your Fabric workspace:  
`https://app.fabric.microsoft.com/groups/<WORKSPACE_ID>/...`

### Step 4 вҖ” Create Fabric SQL Database + seed data

```powershell
.venv\Scripts\python.exe setup_fabric_sql.py
```

This will:
- Create a Fabric SQL Database named `poc_translytical_v3`
- Create 4 tables: `dbo.Products`, `dbo.StatusHistory`, `dbo.ProductAnnotations`, `dbo.PriceHistory`
- Seed 10 Microsoft products + 13 status history rows
- Write `fabric_sql_config.json` with the DB connection details

### Step 5 вҖ” Deploy Semantic Model + Report

```powershell
.venv\Scripts\python.exe create_dashboard_v3.py
```

This deploys:
- **Semantic Model** (`poc_v3_dataset`) вҖ” DirectQuery to Fabric SQL DB
- **Report** (`Translytical_POC_Dashboard_V3`) вҖ” with buttons, slicers, and action layout

## UDF Setup (Manual вҖ” in Fabric Portal)

### A вҖ” Paste the UDF code

1. Fabric workspace вҶ’ open **`poc_writeback`** UDF вҶ’ **Edit**
2. Select all existing code вҶ’ delete вҶ’ paste full contents of `poc_udf_v3.py`

### B вҖ” Register the connection

1. Click **Manage connections** вҶ’ **Add connection**
2. Type: `SQL Database`
3. Select: `poc_translytical_v3`
4. **Alias**: `pocdb` вҶҗ must match exactly
5. Save

### C вҖ” Add library

1. Click **Libraries** вҶ’ **Add library**
2. Name: `requests` | Version: `2.31.0`
3. Save

### D вҖ” Publish

Click **Publish** and wait for green confirmation.

## Database Schema

```sql
-- Core product table
CREATE TABLE dbo.Products (
    ProductID    INT IDENTITY(1,1) PRIMARY KEY,
    ProductName  NVARCHAR(200)   NOT NULL,
    Category     NVARCHAR(100),
    UnitPrice    DECIMAL(10,2)   DEFAULT 0.00,
    Status       NVARCHAR(50)    DEFAULT 'Active',
    LastUpdated  DATETIME2       DEFAULT SYSDATETIME()
);

-- Status change audit log (Tutorial 2)
CREATE TABLE dbo.StatusHistory (
    HistoryID      INT IDENTITY(1,1) PRIMARY KEY,
    ProductID      INT NOT NULL,
    PreviousStatus NVARCHAR(50),
    NewStatus      NVARCHAR(50),
    UpdatedBy      NVARCHAR(200),
    Notes          NVARCHAR(MAX),
    ChangedAt      DATETIME2 DEFAULT SYSDATETIME()
);

-- Free-text annotations (Tutorial 1)
CREATE TABLE dbo.ProductAnnotations (
    AnnotationID INT IDENTITY(1,1) PRIMARY KEY,
    ProductID    INT NOT NULL,
    Annotation   NVARCHAR(MAX),
    CreatedAt    DATETIME2 DEFAULT SYSDATETIME()
);

-- Price change audit log (Tutorial 1)
CREATE TABLE dbo.PriceHistory (
    PriceHistoryID INT IDENTITY(1,1) PRIMARY KEY,
    ProductID      INT NOT NULL,
    OldPrice       DECIMAL(10,2),
    NewPrice       DECIMAL(10,2),
    ChangedAt      DATETIME2 DEFAULT SYSDATETIME()
);
```

## UDF Functions Reference

### `update_product_status`
Writes a status change to `dbo.Products` + logs to `dbo.StatusHistory`. Optionally posts a Teams Adaptive Card.

**Allowed status values**: `Active`, `Low Stock`, `Discontinued`

### `request_status_update`
Sends a Teams notification requesting a status change. No DB write вҖ” read-only lookup.

### `add_annotation`
Inserts a free-text annotation into `dbo.ProductAnnotations` for the selected product.

### `update_price`
Updates `dbo.Products.UnitPrice` and logs the old/new price to `dbo.PriceHistory`.  
Validation: price must be between `$0` and `$100,000`.

## Known Issues & Troubleshooting

| Issue | Cause | Fix |
|---|---|---|
| "Failed to test connection" in SM settings | SM credentials not set | SM вҶ’ Settings вҶ’ Data source credentials вҶ’ Edit вҶ’ OAuth2 вҶ’ Sign in |
| "Error fetching data" on slicer visuals | Slicers not bound to UserInputs columns | Configure each slicer's Field binding in Edit mode |
| `User Data Function` not in Action dropdown | UDF not published yet | Publish the UDF first, then configure buttons |
| DB name reservation error after delete | Fabric reserves names for ~5 min after deletion | Wait or use a different name |
| `alias` not accepted with special chars | Fabric UDF alias field restricts characters | Use lowercase alphanumeric only (e.g. `pocdb`) |
